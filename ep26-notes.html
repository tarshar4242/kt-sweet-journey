知行合一 Manus AI：Skill 與 Manus Agents

EP26 談 Manus AI 的兩個進階主題：一是「Skill 技能包」——把最佳實踐、工作流程與工具打包成可重複呼叫的技能；二是「Manus Agents」——擁有獨特身份、持久記憶、可在 Telegram 等聊天應用裡替你連續完成多步驟任務的個人 AI 代理。課堂實作了用 skill-creator 從零打造一個「資訊圖表 Prompt 產生器」技能。

CHAPTER 01

1. Skill 是什麼與元素

Skill ＝ 為你的智能體（Agent）提供「預封裝、可重複」的最佳實踐與工具。

簡單說，Skill 就是一個技能包：把某件事「該怎麼做」——包含知識、工作流程、工具整合——事先包好，之後只要用關鍵字觸發，Manus 就能照著標準流程把事情做出來。

一個 Skill 由這些元素組成

  • MD 文件（SKILL.md）：技能的說明書——定義這個技能是什麼、運作流程與規範。
  • scripts 腳本：實際執行工作的程式碼／腳本。
  • 資源（references／templates）：技能會用到的參考資料與模板。
  • 觸發邏輯與關鍵字：用什麼關鍵字可以喚起這個技能。
圖解｜Skill＝說明書（SKILL.md）＋腳本（scripts）＋資源（references／templates）＋觸發關鍵字，四者打包成一個可重複呼叫的技能包。
CHAPTER 02

2. 剖析 Skill 製作流程

課堂用「五步」帶大家看懂一個 Skill 從無到有、再到能被閱讀呼應的完整過程。

  1. 了解 Skill 元素：先認識一個技能包由 MD 文件、scripts 腳本與資源組成。
  2. 創作 Skill：用 skill-creator 這個「核心技能創建工具」開始打造新技能。
  3. 剖析 Skill 製作過程：觀察 AI 如何規劃、開發、撰寫文件、交付。
  4. 下載 Skill 檔案：把完成的技能包（ZIP）下載下來檢視。
  5. 仔細閱讀並呼應：打開 SKILL.md 與 scripts，理解技能內容並與需求對照。
⚠️ 重點：不要把 Skill 當黑盒子。下載後親自打開 SKILL.md 讀一遍，才知道它到底怎麼運作、觸發後會做什麼，之後才好維護與修改。
CHAPTER 03

3. Manus vs Claude Code

兩者都能執行技能與程式，最大的差異在於「執行」發生在哪裡、由誰承擔風險。

比較面向ManusClaude Code
核心定位在安全沙盒環境中端到端執行技能在本地端執行程式與指令
執行方式安全的雲端虛擬機本地機器（具潛在風險）
使用環境基於網頁操作，無需額外設置需安裝本地 IDE／CLI 環境
技能導入一鍵即時導入需手動設置與配置
安全性隔離式沙盒執行在使用者機器上運行
適用對象一般使用者、非技術背景者專業開發者
「最大的差異在於『執行』。」
—— Manus 在雲端沙盒替你跑（安全、免設定、非技術者友善）；Claude Code 在你自己的電腦上跑（更彈性、但風險與設定都在本地）。
CHAPTER 04 · LIVE DEMO

4. 實作：用 skill-creator 做「資訊圖表 Prompt 產生器」

課堂現場示範：從對話開始，用官方的 skill-creator 技能，打造一個能產出資訊圖表 YAML Prompt 的儀表板網頁技能。

實作流程（四步）

  1. 從對話開始：輸入「幫我使用 /skill-creator 一起創建一個技能。首先問我這個技能應該做什麼。」
  2. AI 確認目標並反問細節：Manus 先讀 /skill-creator 規範，確認要做「資訊圖表 Prompt 產生器」，並反問是否需多語系、有無 YAML 驗證規則。
  3. AI 規劃並開發：讀規範 → 開發網頁（HTML/CSS/JS）→ 撰寫 SKILL.md → 交付成果。
  4. 驗證並交付：產出 dashboard.html＋SKILL.md，打包成 ZIP，按「新增到我的技能」入庫。
🧾 這次實作耗費 96 點。交付內容：SKILL.md（定義流程、YAML 結構、使用說明）＋ dashboard.html（單頁儀表板，含即時預覽）。

做出來的儀表板長怎樣

  • Dashboard 風格介面：左側功能選單、右側設定區、下方輸出預覽。
  • 多樣化控制項：下拉選單、色碼選擇器、字級滑桿等。
  • 即時預覽：調整參數即時更新 YAML。
  • 快速預設：極簡黑白、科技藍、學術灰三種模板。
  • 多格式輸出：YAML、Markdown、JSON、純文字。

SKILL.md 的 YAML 前置設定（frontmatter）長怎樣

--- name: infographic-prompt-generator description: 產生資訊圖表設計的 YAML Prompt。此技能提供一個互動式 　　　　　儀表板，讓使用者透過視覺化介面選擇參數，即時生成 　　　　　符合特定結構的資訊圖表 Prompt。 license: Complete terms in LICENSE.txt ---
⚠️ 每個 Skill 的 SKILL.md 開頭都有這段 YAML frontmatter：name（技能代號）＋description（描述，也是觸發判斷的依據）。描述寫得越清楚，Manus 越知道什麼時候該喚起這個技能。
CHAPTER 05

5. 官方技能庫與應用篇

Manus 內建一個「技能」頁面，可搜尋、新增自訂技能，也有多個官方技能可直接用。

幾個官方技能

技能用途
skill-creator（技能創造者）核心技能創建工具：用於建立或更新技能的指南，透過專業知識、工作流程或工具整合擴充 Manus。
stock-analysis用金融市場數據分析股票與公司：公司概況、技術見解、價格圖表、內部人持股、SEC 資料。
video-generator專業 AI 影片製作工作流程：影片、短片、廣告等內容。
similarweb-analytics用 SimilarWeb 流量數據分析網站與網域：流量指標、參與度、全球排名、流量來源。

應用篇：把 Skill 用在動畫製作

  • 應用篇 1：Remotion Skill——製作轉場動畫。
  • 應用篇 2：Manim Skill——製作科學原理演示動畫。
💡 Remotion＝用程式碼（React）做影片動畫；Manim＝數學家 3Blue1Brown 那種「公式一步步演示」的動畫引擎。把它們包成 Skill，就能用一句話喚起整套製作流程。
CHAPTER 06

6. Manus Agents 個人代理

從「工具」升級成「有身份的助理」：認領一個屬於你、會跟你一起成長的個人 Agent。

四大特點

特點說明
獨特的 AI 身份為它命名並賦予個性，它會記住所有內容。
持久的記憶與電腦24/7 雲端助理，保留全部上下文與記憶。
自訂技能為助理配備特定領域的專業知識（就是前面的 Skill）。
在聊天應用中使用可在 Telegram 中使用，更多通訊應用即將上線。

可在哪裡使用

  • 已上線：Telegram、電腦使用。
  • 即將上線：WhatsApp、Messenger、Line。
  • 帳號：連結你的 Manus 帳號即可（免費／付費帳號都能用）。
🖥️ 在聊天介面就能直接指派多階段工作流程，無需跳轉頁面；支援語音輸入與檔案上傳，並可即時取得成果；提供 Manus 1.6 Max（深度推理）與 Manus 1.6 Lite（快速回應）兩種模型，可依需求切換。
CHAPTER 07

7. 傳統 Chatbot vs Manus Agents

同樣在聊天視窗打字，差別在於：一個只會「回答」，一個會「把任務做完」。

比較面向傳統 ChatbotManus Agents
核心定位問答型互動工具任務導向型 AI 代理
互動方式一問一答可下達多步驟任務
任務處理單一回合回覆為主可連續執行、拆解並完成複雜任務
資料整理提供文字說明主動整理資料、研究主題、彙整重點
專案規劃協助提供建議生成專案規劃、步驟流程與執行方案
內容生成依提示產出內容整合資料後產出完整成果
檔案處理文字為主可處理文件、圖片、語音等多媒體
輸入形式文字輸入文字、語音訊息、檔案上傳
輸出結果單次回應內容任務成果、整理文件、研究報告等完整輸出
使用價值提升回應效率建立可執行、可交付成果的 AI 工作流
「Chatbot 給你答案，Agent 給你成果。」
—— Chatbot 是一問一答的回應；Agent 會拆解任務、連續執行、最後交付一份可用的成品。
CHAPTER 08

8. Manus Agents 的 Telegram 用法與九大功能

Telegram 上的四大特性

  • 多步驟任務執行：下一個任務，Agent 連續執行並把結果回傳。
  • 多媒體互動處理：語音、照片、文件都能互動。
  • 個性化回應風格：結構化或對話性，依你設定的個性回覆。
  • 彈性模型切換：Max（深度推理）／Lite（快速回應）自由切換。

可完成的九大功能

功能說明
1. 查找資料上網搜尋、蒐集新聞與資訊。
2. 撰寫文書產出文件、DOC 等文書內容。
3. 整理分析數據整理、表格彙整。
4. 建置網站建站並部署上線。
5. 製作簡報產出 PDF／PPTX 簡報。
6. 編寫程式寫程式、執行腳本。
7. 處理檔案處理 PDF、JPG 等各類檔案。
8. 排程提醒定時辦事、按時執行提醒。
9. 影像製作生成圖片與影像。
🍌 課堂實例：在 Telegram 對 Agent 說「用 Nano Banana Pro 做一張 Podcast 封面（EP2｜AI 素養＝未來競爭力？）」，約 3～5 分鐘後 Agent 回傳 2048×2048 正方形封面（深藍科技風＋AI 晶片大腦主視覺），還能繼續調顏色、字型、版面或加 Logo。
🗺️ 平台擴展路線圖：目前已開放訂閱使用，未來將擴展至 WhatsApp、LINE、Slack、Discord 等更多通訊平台。

9. 重點整理（帶走這幾句）

  • Skill ＝ 可重複呼叫的技能包：由 SKILL.md（說明書）＋scripts（腳本）＋資源＋觸發關鍵字組成。
  • skill-creator 是「做技能的技能」：用一句話就能請 Manus 幫你從零打造新 Skill（本集實作耗 96 點）。
  • Manus vs Claude Code：差在「執行」——Manus 在雲端沙盒（安全、免設定），Claude Code 在本地（彈性、風險自負）。
  • Manus Agents ＝ 有身份、有記憶的個人代理：命名、賦予個性、配自訂技能，24/7 雲端待命。
  • Chatbot 給答案，Agent 給成果：Agent 能拆解並連續完成多步驟任務。
  • Telegram 就能用：多步驟任務、多媒體、個性化、模型切換，涵蓋查資料到影像製作九大功能。
