STEP 01

什麼叫「純前端」小工具

先弄懂這個詞在講什麼，你才知道自己適不適合做。

發生什麼事

我以前一聽到「做一個工具」，腦中就浮出一堆讓我卻步的關鍵字：要架伺服器、要開資料庫、要買主機。後來我才發現，我真正需要的東西根本不用這些。整個工具可以只是一個 HTML 檔，你打開瀏覽器，它就能動，不用安裝、不用註冊主機、不用付錢租空間。這種做法有個名字，叫「純前端」，白話講就是：所有事情都在你打開的那個網頁畫面裡發生，沒有另外一台機器在背後幫你算。

我學到什麼

不是每個工具都要一台伺服器，是大部分我想要的小工具，一個檔案就夠。
先弄懂這個詞，你會發現自己早就跨過了那道門檻

可複製的做法

🍀 下次想做一個小工具，先問自己一句：「這個東西，打開瀏覽器就能直接用嗎？」如果答案是可以，你要的多半就是一支 HTML 檔，不用先去想什麼後端、什麼資料庫。
STEP 02

什麼需求適合做成小工具

不是所有需求都適合純前端，這裡是我自己抓出來的判斷點。

發生什麼事

我把手上想做、或已經做出來的東西攤開來看，抓到三個判斷點：

  • 這個東西是我自己用，還是要給很多人同時看到同一份資料？（自己用最適合）
  • 資料要不要長期留著、給下次的我或別人看？（大多只需要暫存在這台裝置的瀏覽器裡就夠）
  • 有沒有現成、公開、免費的資料可以直接串進來？（有的話，工具馬上就有真實內容，不是空殼）

台股回測工具跟 QR code 產生器，剛好都符合「自己用、不用長期共享一份資料庫」這個條件，所以純前端做起來剛剛好。

我學到什麼

不是工具越複雜越厲害，是需求越單純，越該做成一個檔案就好。
先看需求的大小，再決定要不要動用複雜的技術

可複製的做法

🍀 把你手上想做的小工具列一張清單，每個都問自己「這是我自己用，還是要很多人一起用同一份資料」。多半答案是「自己用」，那就是純前端最適合出手的位置。
STEP 03

怎麼跟 AI 開口要一個工具

不用懂任何程式語言的名字，你只要會講一句人話。

發生什麼事

我不會寫程式，但我發現只要把「我要什麼」講清楚，AI 就能把工具生出來。像 QR code 產生器，我跟 AI 說的大概就是這樣一句話：

「我要一個網頁，貼上一個網址，就產生一張 QR code 圖，可以下載存成圖檔。」

我沒有講任何一個程式語言的名字，也沒有講任何技術規格，就只是把「輸入什麼、按了什麼會發生什麼、結果長什麼樣」講清楚。

我學到什麼

你不用會寫程式，你只要會把需求講成一句人話。
越具體，AI 給你的成品越接近你要的樣子

可複製的做法

🍀 開口跟 AI 要工具時，講清楚三件事：輸入什麼、按了什麼會發生什麼、結果要長什麼樣。這三件事講清楚，比會不會用術語重要一百倍。
STEP 04

串公開資料：API 是什麼的白話版，台股回測實例

「串 API」聽起來很技術，其實是跟別人說好一句暗號。

發生什麼事

台股回測工具會去抓股票的歷史價格，這個「去別人那邊拿資料」的動作，術語叫「串 API」。白話講就是：網路上有些單位把資料整理好放著，你的網頁打個招呼（送一個網址，附上一把通行證），對方就把一份乾淨的資料送回來。我的回測工具串的是一個叫 FinMind 的公開資料源，拿到股價之後，接下來算策略、畫圖表全部在瀏覽器裡自己完成，資料完全不經過我自己的伺服器，因為我根本沒有伺服器。

我學到什麼

串 API 不是什麼高深技術，是跟別人說好一句暗號，資料就送過來。
技術名詞聽起來嚇人，拆開看常常就是一句話的事

可複製的做法

🍀 想讓工具有真實內容而不是空殼，先問 AI：「這個主題有沒有公開、免費的資料可以查？」通常真的有，你要做的只是把那份規則抄一次給 AI，剩下讓它去處理。
STEP 05

借現成的功能，不用自己重寫

別人已經寫好、願意公開讓大家用，你只要負責掛上去。

發生什麼事

QR code 產生器裡，「怎麼把一個網址變成一張 QR code 圖」這段邏輯，我完全沒有自己想過。因為網路上早就有人把這段邏輯寫好、免費公開讓大家用，這種東西有個名字叫「函式庫」。我要做的只是在我的網頁裡掛一行，告訴瀏覽器「去載這個別人寫好的功能」，剩下的排版、加下載按鈕，才是我自己動手的部分。

我學到什麼

不是每個功能都要自己生出來，是先看有沒有人已經做好，願意借你用。
會借東西，跟會做東西一樣重要

可複製的做法

🍀 做小工具卡關時先問 AI：「這個功能有沒有現成、免費、別人已經寫好的東西可以直接用？」常常真的有，你只負責排版跟串起來，不用從零發明一次。
STEP 06

部署分享：丟上 GitHub Pages 就是網址

工具做在自己電腦裡，只有自己看得到，要分享得再做一步。

發生什麼事

工具寫好放在自己電腦裡，別人打不開。要讓它變成一個誰都能點開的連結，跟我做論文研讀庫那次一樣，一樣是那三行指令：

git add . git commit -m "更新工具" git push origin main

打完這三行，兩分鐘後就變成一個網址，不用密碼、不用付費、不用管主機。我手上現在就有兩個這樣活著的網址：

  • tarshar4242.github.io/give-me-money（台股回測工具）
  • tarshar4242.github.io/renmei-qrcode-generator（QR code 產生器）

我學到什麼

一個小工具，做完不分享，等於只做了一半。
部署這一步，門檻比想像中低很多

可複製的做法

🍀 工具做完別急著想「夠不夠好」，先丟上 GitHub Pages 變成一個網址，自己先用個幾天。有問題再回頭改，反正修改一樣是那三行指令，不用怕動了就壞掉。
STEP 07

工具箱思維：不是每個都能純前端到底

誠實面對純前端的邊界，遇到限制加一點點最輕量的幫手就好。

發生什麼事

純前端很方便，但不是萬能。我另外做過一個下載 YouTube 字幕的小工具，一開始想直接在瀏覽器裡跟 YouTube 要資料，結果瀏覽器基於安全限制不給這樣做（這叫「跨域限制」，白話講是網站之間互相設了門檻，不能隨便跨過去拿別人的資料）。這個工具後來多加了一支很輕量的本機小程式幫忙開門：網頁負責操作畫面，那支小程式負責去跟 YouTube 要資料，兩個檔案一起動才能用。它不算完全的純前端，但也遠遠沒有到「要架一整套後端系統」那麼重，開電腦跑一行指令就能動。

我學到什麼

不是所有工具都能純前端到底，是遇到限制時，加一點點最輕量的幫手就好，不用整套升級成大型系統。
先誠實面對限制，再找最小的解法

可複製的做法

🍀 遇到「瀏覽器不給做」的狀況，先別自己亂猜是哪裡壞了，把狀況講給 AI 聽，問它：「這是瀏覽器的限制嗎？有沒有最輕量的解法？」通常會有一個比「整套架伺服器」小得多的答案。
