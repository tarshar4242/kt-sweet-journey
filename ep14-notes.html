CHAPTER 01

1. 什麼是 Open Notebook

核心定位

Open Notebook 是一個開源、AI 驅動的筆記與研究平台，結合 AI 力量與隱私控制。它是 Google NotebookLM 的開源替代，為研究者、學生與專業人士設計，讓使用者在增強能力的同時，掌握自己的工作流程與資料使用方式。

核心理念

在 AI 主導的時代，獲取知識與思考能力不應是少數人的特權，也不應被鎖在單一供應商的服務中。

CHAPTER 03

3. Open Notebook 解決的七大痛點

  • 資料隱私疑慮：敏感資料不外流，完全掌握安全。
  • 內容轉換受限：支援多元知識萃取與轉換。
  • AI 供應商綁定：自由切換模型，不受單一限制。
  • 缺乏 API：完整 API 整合自動化流程。
  • 功能限制：開源架構支援無限擴充。
  • Podcast 限制：支援 4 人對話與個性自訂。
  • 成本不透明：自主選擇模型，成本完全可控。

最低需求推薦配置

支援作業系統：Linux（官方推薦，效能最佳）／macOS（Intel 或 Apple Silicon）／Windows（需搭配 WSL2 或 Docker Desktop）。

三種安裝方式

  1. Docker Compose（推薦）：最簡單快速的部署方式，自動處理所有依賴，適合大多數使用者。
  2. Docker 指令安裝：提供更靈活的配置選項，適合熟悉 Docker 操作的進階使用者。
  3. 源碼安裝 (Source)：完全控制安裝過程，適合開發者或需要深度客製化的使用者。
CHAPTER 08

8. 實戰安裝：Docker Compose 手把手教學

整理自課程附帶的《Open Notebook 安裝操作手冊》（共 7 頁），以 Windows + Docker Desktop 為例。

一、前置條件檢查

  • 安裝 Docker（docker.com），確認：docker --version 與 docker compose version（或 docker-compose --version）。
  • 開啟 Docker Desktop → Settings → General：勾選 Use WSL 2 based engine；Resources → WSL Integration：勾選 Enable integration with default WSL distro（例如 Ubuntu-24.04）。
  • 回到主畫面確認右下角顯示 Engine running——這步沒完成，後續指令會全部失敗。

二、建立專案資料夾與設定檔

  1. 開啟 PowerShell，建立並進入資料夾：mkdir D:\work\open-notebook、cd D:\work\open-notebook（路徑可自訂，但後續指令要在同一資料夾內執行）。
  2. 建立 docker-compose.yml：notepad docker-compose.yml，把官方提供的內容完整貼上並覆蓋，只修改 OPENAI_API_KEY，其餘不要改。
  3. 各大 LLM API 申請位置：OpenAI（platform.openai.com/api-keys）／Anthropic（console.anthropic.com）／Google（aistudio.google.com）／Groq（console.groq.com）。

三、啟動與驗證

  1. （保險）先關閉舊服務：docker compose down（若沒裝過可跳過）。
  2. 啟動服務：docker compose up -d。
  3. 確認狀態（非常重要）：docker compose ps，正常應顯示 open-notebook-surrealdb-1 → Up、open-notebook-open_notebook-1 → Up，開放埠 8000／8502／5055。若顯示 Restarting 或 Exited，先別繼續下一步。
  4. 開啟瀏覽器輸入 http://localhost:8502：正常應可看到 UI、可建立 Notebook、可正常對話，且沒有 ERR_EMPTY_RESPONSE 或 Failed to send message。
常用管理指令停止服務　docker compose down 啟動服務　docker compose up -d 查看狀態　docker compose ps 查看 Open Notebook log　docker compose logs --tail=100 open_notebook 查看 SurrealDB log　docker compose logs --tail=100 surrealdb
💡 若想「資料不出電腦、也不用申請其他 LLM API」，可改用 Ollama 跑本地模型（課程提及後續直播會另外分享）。安裝輔助小幫手：官方提供的 ChatGPT 安裝助理連結（見簡報／手冊原文）。
CHAPTER 09

9. 建立筆記本＋自動化知識轉化

創建第一個筆記本

  1. 點擊主畫面右上角「Create New Notebook」。
  2. 填寫名稱與詳細描述——描述越詳細，AI 越能理解上下文（例：「這個筆記本專門用於收集和整理深度學習相關的論文、教學影片和實作範例…」）。
  3. 點擊「Add Source」加入資料：連結（Link，貼 URL）／檔案（File，PDF/DOCX/PPT）／文字（Text，直接貼上）。處理流程：系統解析 → 提取純文字 → 建立向量索引。

自動化知識轉化 (Transformations)

Transformations 是預設的 AI 指令集，能自動處理匯入的資料來源，節省大量閱讀與整理時間。

  • 自動摘要：快速生成長篇文章或論文的重點摘要。
  • 實體提取：自動抓取人名、地點、日期與專有名詞。
  • 語言翻譯：外文資料直接轉化為母語。
  • 格式轉換：非結構化文字轉為表格、Markdown 或心智圖代碼。
💡 進階技巧：可自行編寫 Prompt 創建客製化 Transformation，例如「提取所有提到的書籍並附上作者」。
CHAPTER 10

10. 與 AI 助手對話的三種範圍＋打造 AI Podcast

對話範圍適用情境
單一文件 (Single Source)深入研讀論文、合約或長篇文章
筆記本 (Notebook)｜預設整合當前筆記本內所有資料，適合特定主題綜合研究
全域 (Global)搜尋所有筆記本與資料庫，適合跨領域整合或廣泛檢索

Pro Tip：回答不夠精準就縮小到「單一文件」；需要創意發想就切到「全域」。

打造專屬 AI Podcast（多角色對話）

  1. 選擇來源：勾選要轉換的筆記本或特定文件。
  2. 設定參數：語言（中文／英文）、對話風格（輕鬆／專業）、主持人數量（最多 4 位）。
  3. 生成播放：AI 自動撰寫腳本並合成語音，完成後可線上收聽或下載。

11. 隱私控制三層級＋誰最需要 Open Notebook

層級內容
資料儲存自主權所有筆記與文件存在本地伺服器或私有雲端，不依賴第三方資料庫
AI 訪問控制精細設定 AI 可讀取範圍（單一文件／特定筆記本／全域）
模型隱私選擇可用 Ollama 等本地 LLM，實現完全離線運算

六大核心族群

  • 學術研究員：整理大量文獻、精準引用、深度分析與知識連結。
  • 內容創作者：靈感快速轉化為 Podcast 腳本或文章。
  • 技術開發者：喜愛開源專案、想完全掌控架構與 API 整合。
  • 隱私倡議者：拒絕資料上雲，堅持數據主權與本地部署。
  • 學生與考生：快速摘要教材重點，製作複習筆記與心智圖。
  • 知識工作者：整合跨專案資訊，建立個人第二大腦。
💡 實際案例：學術研究整理數百篇 PDF 文獻自動生成綜述；企業知識庫匯入內部規章打造員工 AI 問答助手；個人學習快速摘要課程內容；Podcast 製作把零散筆記轉成廣播腳本並自動生成語音。
