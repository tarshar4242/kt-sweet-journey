OVERVIEW

1. 一頁看懂全課程

這堂課的骨架非常清楚：一個核心問題（2026 上半年，哪些 AI 能力已經是「標配」？）＋四條技能主線。四章不是並列的工具清單，而是一條由淺入深的路徑：先把「說話」練好（文字），再把「視覺」建成系統（生圖），然後把工作「交辦」出去（Agent），最後讓一切「自己跑」（自動化）。

「工具每季都在換，能力的分層不會換：會說、會看、會交辦、會自動。」
—— 本課四章背後的共同邏輯
CHAPTER 01

2. 第一章　✍️ 文字篇

文字是所有 AI 應用的地基。這一章的主軸：把「碰運氣的問法」升級成「有結構的系統」。

2.1 提示工程三段進化：Prompt → Context → Harness

  • L1 Prompt：四要素缺一不可——「你是誰（角色）、做什麼（任務）、長怎樣（格式）、不能怎樣（限制）」。先寫格式再寫任務，AI 就不會自由發揮。
  • L2 Context：把滿意的舊回答存下來當範例（few-shot）、附上原始資料與術語表，讓 AI「進入狀況」再動筆。
  • L3 Harness：同類任務問超過三次就模板化——存成 Gem／GPTs／專案自訂指令，加上「產出後自我檢查」的固定收尾。

課堂示範指令模板

【角色】你是一位資深行銷文案編輯 【任務】把下面的產品說明改寫成 3 種語氣的社群貼文 【格式】每則 80 字內，附 2 個 hashtag，用表格輸出 【限制】不得出現誇大療效字眼；保留品牌名原文 ---（貼上原始素材）

2.2 YAML 結構化輸出控制

要讓 NotebookLM 資訊圖表、報告排版「乖乖聽話」，關鍵是把需求從形容詞變成欄位。YAML 是「人看得懂、AI 不會誤會」的中間格式，本系列 EP23、EP-YAML 專場都圍繞這招。

  • 五大應用場域：文字規格、圖像風格、資訊圖表、報告編排、工作流定義。
  • 先請 AI「把我的需求整理成 YAML」→ 人工確認欄位 → 再生成，錯誤率大幅下降。
  • 改版只改欄位值、不重寫整段話——版本差異一目了然，團隊也能接手。

2.3 NotebookLM 個人知識庫

  • 把講義 PDF、網頁、影片連結、錄音統統丟進同一個 Notebook 當來源；回答一律附引用，不亂編。
  • 一鍵轉換多種形態：學習指南、簡報大綱、時間軸、語音摘要（播客）、資訊圖表。
  • 與 2.2 連動：用 YAML 規格卡控制它生出來的簡報與圖表版型。
⚠️ 來源品質決定答案品質——上傳前先清掉過時與重複的檔案，是最容易被忽略的一步。

2.4 巨量資料摘要精煉

  1. 長文切塊：依章節或每 3–5 千字切一段。
  2. 逐段用固定欄位摘要（summary／numbers／quotes／todo）。
  3. 把所有段落摘要合併，做最終總結論——Map-Reduce 思維。

摘要一定要帶「任務目的」：給主管的重點、給自己的筆記、給客戶的亮點，是三種不同的摘要。

「先分段消化、再統一格式，最後才做總結論。」

CHAPTER 02

3. 第二章　🎨 生圖篇

上半年生圖最大的觀念升級：從「一張一張抽卡」到「建立可複用的風格系統」。本系列 EP41 的口號說得最傳神——不只給漁獲、不只給釣竿，直接給你一張智慧漁網。

3.1 風格圖卡系統（YAML 風格卡）

style_card: name: 奶油紙張手寫筆記 canvas: 4:5 直式 palette: [奶油白, 焦糖棕, 墨綠] texture: 紙張纖維、鉛筆手寫感 layout: 大標 + 3 重點區塊 + 手繪箭頭 rule: 繁體中文, 重點詞加底線 input: （貼上要做成圖卡的文章）
  • 課堂系列實例：10 款筆記風格圖卡 Gem（貼入文章即生成指定風格圖卡）、寫真風格圖卡（員工大頭貼、活動紀念照置換）。
  • 先讓 AI 分析參考圖反推 YAML，比自己從零描述快得多。

3.2 一致性人物與置換應用

  • 鎖定人物特徵 + 統一視覺規格 → 批次生成整個部門的大頭貼／紀念照，風格整齊劃一。
  • 訣竅在「約束」：鎖臉部特徵、鎖服裝色系、鎖背景元素，其餘才交給模型。
  • 先做 1 張標準樣張給大家確認，再批次跑全部人。
⚠️ 涉及真人肖像：取得本人同意、標示 AI 生成、確認公司規範——內部使用也一樣。

3.3 簡報與資訊圖表視覺化

需求首選工具備註
從資料長出圖表NotebookLM + YAML來源可追溯、版型可控
簡報美化與動畫Canva（Magic layers）拆層動畫、品牌套版（EP34）
一句話生整份簡報Gamma / Prezi AI速度快、細修空間較小（EP27）
自由度最高AI 生圖 + 手動排版風格卡先行，量產最穩

簡報視覺三層工法：版型（layout）→ 圖像（image）→ 動態（motion），一層一層加，不要一次全包給 AI。

3.4 影片生成入門與 API 批量

  • 影片生成（Google Video／Flow 級工具，EP43 專題）目前最穩的用法是短素材：開場動畫、產品展示、社群短片。
  • 穩定工法：風格卡先生「首幀圖」→ image-to-video 生 4–8 秒片段 → 剪接成片；重要文字與 logo 後製加。
  • 量產靠 API（如免費的 Pollinations）：「介面生一張，API 生一批」，搭配試算表跑迴圈自動出圖。
「抽卡靠運氣，風格卡靠系統。」

4. 第三章　🤖 AI Agent 篇

上半年最大的分水嶺：AI 從「回答你」變成「幫你做完」。這一章從觀念分層講到實際上手路徑。

4.1 Agent 是什麼：從聊天到代理

4.2 Claude Code：終端機裡的工程代理

  • 直接讀寫專案檔案、執行指令、跑測試、發 PR——本系列 EP32 專題實測過的主力工具。
  • CLAUDE.md 是專案說明書：把規範寫進去，每次對話自動遵守。
  • Skills／斜線指令：把常用流程做成技能包，重複任務一鍵呼叫。
  • 最佳實踐：先計畫再動手、小步提交、要求它自己驗證成果。
  1. 第一步交辦「幫我看懂這個專案」，讓它先摘要架構。
  2. 給一個小任務（改文案、加一頁），看它的計畫再放行。
  3. 滿意的流程沉澱成 CLAUDE.md 規範或 Skill。
✅ 實例：本份筆記與搭配的互動技能地圖網頁，正是用「交辦式」流程請 AI Agent 從雲端講義自動產出的成品。

4.3 OpenCode：不花大錢的 Agent 體驗

本系列 6/26 加開場專題。開源終端 Agent 介面與商用工具相近，模型可自選——接免費或低價 API 就能練「交辦、拆解、驗收」的肌肉，能力可遷移到任何 Agent 工具。

4.4 MCP 與工具生態

  • MCP（Model Context Protocol）＝AI 連外部工具的通用插座：接上 Drive、Gmail、Notion、資料庫，Agent 才有手有腳。
  • 權限由小到大：先接讀取類（查資料），再接寫入類（改東西）。
  • 寫入權限（寄信、刪檔、付款）務必保留人工確認那一步。

4.5 Vibe Coding：把想法直接說成網站

本集的壓軸示範——課程簡報首頁就叫 index.html：不寫程式，用「描述＋迭代」做出多層次互動學習網頁（參考典範：akira.ipas-ai.com 的互動式 AI 學習地圖）。單檔 HTML 免安裝、免後端，丟上 GitHub Pages 就能分享。

請幫我做一個單檔 HTML 學習地圖： 1. 深色科技風、霓虹漸層、卡片式首頁 2. 四大章節，點卡片進入章節頁（麵包屑導航） 3. 章節內手風琴展開每個技能：重點/步驟/範例 4. 全站關鍵字搜尋 + 學習進度打勾（localStorage） 5. 手機優先、繁體中文

5. 第四章　⚙️ 自動化篇

技能學完，最後一哩是「讓它自己跑」。所有自動化都是同一句話：「當…發生，就用 AI 做…，然後把結果放到…」。

5.1 自動化思維：觸發 → 處理 → 輸出

5.2 n8n 工作流入門

  • 視覺化拉節點連線：表單進來 → AI 分類 → 寫入試算表 → 發通知；AI 節點可接多家模型，HTTP 節點萬用。
  • 可自架（免費）也有雲端版，中文社群資源豐富——也是學員許願最多的主題之一。
  1. 選一個「每天都在做的搬運」：如把客戶來信整理進表格。
  2. 節點三顆就好：Trigger（Gmail）→ AI（摘要＋分類）→ Sheets（寫入）。
  3. 跑順一週後，再加通知與例外處理。

5.3 Google Apps Script 與擴充功能

Workspace 內建的免費自動化。搭配 AI 寫 code：描述需求 → 貼上 Apps Script → 授權執行，全程不用會 JavaScript。進階可打包成擴充功能分享團隊（EP17「從零到上線」路線）。

幫我寫 Google Apps Script： 每天 08:00 讀取「客服回報」試算表新增列， 用 AI 產生 50 字摘要與急迫度(高/中/低)， 回寫到同列 E、F 欄；急迫度=高就寄信通知我。

5.4 會議紀錄流水線與排程代理

  1. 錄影／錄音 → 自動轉逐字稿（含時間戳，EP29 工作流）。
  2. AI 三段式整理：決議、待辦（負責人＋期限）、爭點。
  3. 自動存檔知識庫＋寄給與會者——散會 10 分鐘內送達。
  • 排程代理：讓 Agent 每天固定時間醒來巡邏（收件匣、報表、網站狀態），有異常才叫你。
  • 信件漏斗：名單 → 分眾 → AI 個人化信件 → 自動寄送追蹤；人負責決策與例外，AI 負責重複與整理。
