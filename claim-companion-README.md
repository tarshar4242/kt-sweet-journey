# 理賠陪跑主題館 · claim-companion

> 版本 v1.0（2026-07-23）｜獨立入口頁，掛在課程筆記館「生活應用」系列。
> 專案定位：**不是查詢系統，不儲存、不經手任何個資**。它是一位「陪跑教練」，帶學員在自己的手機上一步一步完成理賠準備。

## 這是什麼

一頁式（單檔多錨點）的操作教學網站，帶 35 到 60 歲以上、對 3C 沒把握的學員，從查醫療紀錄到正式申請理賠，共八個站點，全程「教你動手做」而非「提供資訊查」。

## 結構（單檔多錨點）

主檔 `claim-companion.html` 內含：

- `#home` 入口分流頁：三扇門大按鈕、安心承諾、八站旅程地圖、常見問題入口
- `#s1`～`#s8` 八個站點：每站固定「五段式模板」（① 這一步要做什麼 → ② 為什麼 → ③ 拿手機跟著做 → ④ 打勾清單 → ⑤ 卡住了怎麼辦）
- `#faq` 常見問題：10 張翻卡

八站與常見問題的**內容都寫在檔案底部的 JavaScript 資料區**（`STATIONS` 陣列與 `FAQ` 陣列），改文字只要改那兩個陣列即可，版面會自動套用。

## 技術說明

- 純靜態 HTML + CSS + 原生 JavaScript，無框架、無外部 API、無追蹤、無表單。
- 字體採系統字堆疊（`Noto Sans TC` / `PingFang TC` / `Microsoft JhengHei`）：Android 系統本身就是 Noto、iPhone 用蘋方，皆為乾淨中文字，**刻意不載入 Google Fonts**，以符合「不呼叫外部服務」的隱私原則。
- 打勾清單為**純視覺、不記憶**（重整頁面會清空）——依規格確認選項採行，未使用 localStorage、不上後端。
- 一鍵複製使用 `navigator.clipboard`，並在舊瀏覽器自動退回 `execCommand('copy')`。

## 設計規範

- 色票：底色奶油米白 `#FBF7EF`、主色鼠尾草綠 `#7A9B7E`、深文字暖深灰 `#3E4640`、點綴柔杏桃橘 `#E8A87C`、卡片暖白 `#FFFFFF`。
- 無障礙：內文 18px、行高 1.8、標題 28px 以上、按鈕高度 56px 起、單欄手機優先、桌機內容欄最寬 720px 置中、對比符合 WCAG AA、無輪播無彈窗無 hover 依賴。

## 怎麼改內容

1. **改站點文字**：打開 `claim-companion.html`，找到底部 `const STATIONS = [ ... ]`，改對應站的 `what` / `why` / `steps` / `checks` / `stuck` 欄位。
2. **改常見問題**：找到 `const FAQ = [ ... ]`，改題目 `q` 與答案 `a`。
3. **換真實截圖**：照 `screenshots-needed.md`，在檔案裡搜對應編號（如 `S1-03`），把該示意框換成圖片。
4. 改完 push 到 `main`，GitHub Pages 會自動更新。

## 合規

每頁頁尾固定顯示免責聲明；第 8 站「去哪裡問」只引導學員去問對的人（醫師、藥師、業務員），不做任何醫療判斷或保險條款解釋的結論。
