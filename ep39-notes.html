NotebookLM 與 Markdown 格式
交互應用

同樣一段內容，用自然語言叫 NotebookLM 產出報告——退格、表格常常一匯出就跑掉。這堂課教你改用 Markdown／YAML 格式指令把版面「講清楚」、用工作室預覽複製保住格式，再用 Google Docs＋Apps Script 補上目錄、頁碼與統一字體。本頁依官方課程簡報與課堂 GAS 程式碼整理。

1. 一頁看懂全課程

「NotebookLM 負責『內容與邏輯』；『封面、目錄、頁碼、版面』交給 Google Docs／Apps Script。分工清楚，排版才不會亂。」
—— 本集核心分工
CHAPTER 01

2. 自然語言的坑：退格與表格會不見

先看講師的示範主題：請 NotebookLM「介紹戰鬥陀螺」，要求標題、文字約 400 字、含標題的表格。（簡報 p1–p5）

  • 對話視窗裡看起來都對：有標題、400 字、每段第一句退 2 格、表格也在。
  • 陷阱一（退格）：對話裡雖然「可退格」，但貼到 Word 會恢復原樣（縮排消失）。
  • 陷阱二（報告功能）：改用 NotebookLM 的「報告功能」匯出時——表格不見了、退格也不見了。
  • 結論：自然語言能把「內容」講清楚，卻很難把「格式」穩定帶到最終檔案。
⚠️ 這就是為什麼要換一種「機器看得懂格式」的講法——Markdown／YAML。
CHAPTER 02

3. 用 Markdown／YAML 把格式「規格化」

同一個戰鬥陀螺需求，改用結構化格式來下，NotebookLM 就知道你要的每一項規格。（簡報 p6–p9）

YAML 版：把需求當成一張「規格表」

task: "撰寫戰鬥陀螺介紹" output_format: "markdown" requirements: title: required: true description: "文章標題" content: length: "約400字" formatting: indentation: "每段第一句退2格" text_style: "不可使用加粗字體" topics: "戰鬥陀螺的起源、玩法與媒體發展" table: required: true includes_caption: true description: "戰鬥陀螺的類型與特色比較表"
💡 YAML＝用「鍵：值」層層縮排列出需求，像填一張表單；Markdown＝用 #／##、清單、表格等輕量符號直接寫出結構。兩者都比自然語言更能穩定控制版面。

兩種進階指令版本（簡報 p10 · 自訂筆記本）

版本適合／重點
長文架構與草稿版報告與企劃書：一律 Markdown 輸出、專注「章節架構＋圖表標題預留位置＋內容草稿」；圖表處用 > [圖表插入點：…] 標示；只給乾淨結構化文本，封面／目錄／頁碼後續自己排。
模組化輸出版極長篇正式文件：嚴格用 Markdown；把「章節架構草擬」與「正文草稿」分開產出，先給完整骨架（含各級標題與圖表標題預留），再寫正文。
✅ 這樣下指令有兩個好處：產出好用的 Markdown，且明確界定 NotebookLM 的工作範圍，避免它用純文字硬畫出奇怪的封面或目錄，之後轉 Word／Google Docs／LaTeX 排版更順。
CHAPTER 03

4. 工作室預覽：複製最能保真

格式指令下對了，還要用對「取出」的方式。（簡報 p8–p9）

  • 不要從「報告功能」硬匯出（會掉格式），而是直接使用工作室（Studio）的預覽顯示功能。
  • 從預覽畫面複製，是最能保留格式的方式——退格與表格都比較能帶著走。
  • 心法：Markdown 指令（講清楚）＋ 預覽複製（取乾淨），格式才穩。
「同樣的內容，指令用 Markdown、取用走預覽複製——退格和表格才留得下來。」
保真的兩個關鍵動作
CHAPTER 04

5. 目錄與頁碼：三種做法

課堂用「3 頁 A4：第 1 頁目錄、第 2–3 頁內文」測試三種下法。（簡報 p11–p13）

做法一：YAML 描述文件結構

document_profile: type: "Report" total_pages: 3 layout: page_1: { content_type: "Table of Contents", title: "目錄" } page_2: { content_type: "Body Content", elements: ["Main Heading","Subheading","Textual Discourse"] } page_3: { content_type: "Body Content", elements: ["Main Heading","Subheading","Textual Discourse"] } formatting_rules: paragraph: { first_line_indent: "2_characters", alignment: "justify" } typography: { headings: { main: "H1", sub: "H2" } } pagination: { show_page_number: true, start_from_page: 1, position: "footer" }

做法二：Markdown 手動補「縮排與頁碼」

  • 第 1 頁單獨產「目錄章節」；第 2–3 頁依目錄寫 # 大標題、## 小標題與論述。
  • 傳統 Markdown 不支援首行縮進 → 每段開頭手動加兩個全形空格（或 &emsp;&emsp;）。
  • 每頁底部手動加 [第 X 頁 / 共 3 頁] 標記，方便後續匯出。

做法三：交給 Google Docs 內建功能

  • 首行縮進：全選內文 → 格式 → 對齊與縮進 → 縮進選項 → 特殊縮進選「首行」，設 0.85 公分（約 2 字元），切勿手動連按空白鍵。
  • 頁碼：插入 → 頁碼 → 選「頁尾」並勾「顯示總頁數」（如 1/3）。
  • 第 1 頁放「自動目錄」，此頁不放內文論述。
⚠️ 三種做法各有場合：要快、要一次到位 → Google Docs 內建最穩；要純文字可攜 → Markdown；要把規格交給 AI 產生 → YAML。
CHAPTER 05

6. Google Docs＋Apps Script 收尾

最後把重複的排版動作自動化：課堂提供三段 Google Apps Script（GAS）——寫頁碼、自動生目錄、統一字體。（簡報 p14–p15；程式碼取自課堂《GAS code》文件）

① 頁尾寫頁碼（標楷體 10pt 置中）

function writeFooter() { const doc = DocumentApp.getActiveDocument(); const tab = doc.getActiveTab().asDocumentTab(); let footer = tab.getFooter(); if (!footer) { footer = tab.addFooter(); } footer.clear(); const p = footer.appendParagraph("第 1 頁"); p.setAlignment(DocumentApp.HorizontalAlignment.CENTER); p.editAsText().setFontFamily("標楷體").setFontSize(10).setBold(false); doc.saveAndClose(); }

② 自動抓標題產生「手動目錄」

掃描全文，把第一個非空段落（主標題）、「一、二、三…」的章節標題與以「表」結尾的表格標題收集起來，插到最前面成為目錄頁（標楷體、含頁碼點線），並自動清掉舊目錄。核心函式：

function createManualTOCFromDocument() { const doc = DocumentApp.getActiveDocument(); const body = doc.getActiveTab().asDocumentTab().getBody(); removeOldManualTOC_(body); // 先刪舊目錄 const headings = collectHeadingsFromBody_(body); // 收集標題＋頁碼 body.insertPageBreak(0); const tocTitle = body.insertParagraph(0, "目錄"); tocTitle.setHeading(DocumentApp.ParagraphHeading.HEADING1); tocTitle.setAlignment(DocumentApp.HorizontalAlignment.CENTER); let i = 1; headings.forEach(item => { // 標題 …… 頁碼 const p = body.insertParagraph(i++, makeTOCLine_(item.title, item.page)); p.editAsText().setFontFamily("標楷體").setFontSize(12); }); body.insertPageBreak(i); doc.saveAndClose(); } // 抓標題規則：第一段=主標題；「一、二…」=章節；「…表」=表格標題 // makeTOCLine_() 用「.」補點線對齊頁碼
💡 這段等於幫你把 Word 的「自動目錄」在 Google Docs 裡用程式重現：改標題、加章節後重跑一次就更新，不用手動維護。

③ 一鍵全文統一字體（標楷體 DFKai-SB）

function setAllToDFKaiSB() { const doc = DocumentApp.getActiveDocument(); doc.getBody().editAsText().setFontFamily("DFKai-SB"); // 全文改標楷體 Logger.log("已統一設定為標楷體"); }
⚠️ 使用方式：Google Docs → 擴充功能 → Apps Script → 貼上函式 → 執行（首次需授權）。這三段對應課堂示範的「頁碼、目錄、統一字體」三個常見排版收尾動作。
「會重複的排版動作，就寫一次 Apps Script，之後一鍵跑完。」

7. 名詞卡與立即練習

名詞白話正式定義
NotebookLM以你的資料為本的 AI 筆記本Google 的來源接地（grounded）AI，回答限定在你上傳的來源
Markdown用符號寫排版的輕量語言以 #、清單、表格等符號標記結構的純文字格式
YAML用「鍵：值」列規格以縮排鍵值對表達結構化資料的格式，常用來下「規格式」指令
首行縮進每段第一行空兩格段落層級的排版屬性；Google Docs 設 0.85 公分，勿手打空白
工作室預覽NotebookLM 的成品預覽Studio 的預覽顯示；從預覽複製較能保留退格與表格
Apps Script（GAS）Google 文件的自動化程式以 JavaScript 操作 Docs／Sheets 等的雲端腳本平台

立即練習

  1. 用自然語言請 NotebookLM 寫一段含表格的短報告，貼到 Word，看退格／表格掉了沒。
  2. 改用YAML 或 Markdown 指令重下一次，並從工作室預覽複製，比較差異。
  3. 把成品貼進 Google Docs，用內建首行縮進 0.85cm ＋ 頁碼（顯示總頁數）排好。
  4. 進階：把三段 GAS 貼進 Apps Script，一鍵補頁碼、生目錄、統一標楷體。
