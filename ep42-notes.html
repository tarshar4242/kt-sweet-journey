CHAPTER 01

2. API 到底在做什麼

白話：API 像餐廳服務生。你的程式把需求交給 API，API 送到模型，再把答案帶回來。（簡報 p3）

正式：Application Programming Interface 是一套讓不同軟體依約定格式交換請求與回應的介面。常見用途：聊天機器人、文字摘要、資料分類、客服、內容生成與自動化工作流。

開始前準備（簡報 p4）

  1. 安裝 Python，確認能執行 python --version。
  2. 建立專案資料夾，例如 AI_API。
  3. 準備套件：OpenAI、Google Gen AI、Anthropic、requests 等。
CHAPTER 02

3. OpenRouter：一把金鑰連多模型

OpenRouter 是統一模型入口，可從同一 API 選 OpenAI／Gemini／Claude 或其他模型，適合比較成本、速度、品質與用途。（簡報 p5–p9）

申請流程

  1. 註冊 OpenRouter。
  2. 加入 Credits。
  3. 建立 API Key。
  4. 用 chat/completions 或相容端點測試。
  • 比較四個模型：同一個問題送給不同模型，記錄品質、速度、價格與適用工作。
  • Fusion 多模型協作：讓多模型協作或比較答案，提升研究、推理或生成品質；但多次呼叫會增加成本與延遲。
  • 使用方式：查看模型列表與價格 → 選擇或自動路由 → 取得 API 格式 → 用 API Playground 或 Hoppscotch 測試。
CHAPTER 03

4. OpenAI API 申請與 Colab 最小測試

流程：登入 API Platform → 設定帳務 → 建立 API Key → 在 Google Colab 測試。（簡報 p11–p20）

⚠️ 帳號與帳務：ChatGPT 訂閱和 API 計費是不同系統。即使有 ChatGPT Plus，API 仍可能需要另外設定付款或額度。
🔑 建立 API Key：建立後立即安全保存。金鑰通常只完整顯示一次，不能放在公開簡報、聊天、程式碼截圖或 GitHub。

Google Colab 最小測試四步

  1. 開啟 Google Colab 並建立新筆記本。
  2. 安裝官方套件，例如 pip install -q -U openai。
  3. 透過環境變數或 Colab Secrets 載入 Key。
  4. 發送一個最小請求，確認能收到回應。
✅ 成功條件（簡報 p20）：套件安裝完成、Key 可使用、模型名稱正確、請求收到回應。
CHAPTER 04

5. 三類常見錯誤與除錯順序

錯誤可能原因
Incorrect API key providedKey 錯誤、複製不完整，或讀到錯的環境變數
insufficient_quota沒有額度、帳務未設定，或超過上限
ModuleNotFoundError套件未安裝，或執行環境重啟後遺失
⚠️ 除錯順序：先看錯誤類型，再依序檢查 Key、模型名稱、套件、帳務與網路；不要一看到錯誤就重新建立所有東西。

做自己的自由提問版本（簡報 p22）

把固定問題改成輸入變數，讓使用者在 Colab 輸入問題，再把內容送往模型——這是從一次測試走向小工具的第一步。

CHAPTER 05

6. Token 使用量與三項安全設定

Token 使用量（簡報 p23）

Token 是模型處理文字的計量單位。費用通常與輸入 Token、輸出 Token、模型價格及其他工具呼叫有關。應記錄：input tokens、output tokens、total tokens、模型名稱與請求用途。

三項安全設定（簡報 p25）

  1. 重新命名筆記本，保留可理解的版本紀錄。
  2. 使用 getpass()、環境變數或 Secret，不把 Key 寫死在程式碼。
  3. 設定使用量或預算通知，避免意外成本。

7. Gemini／Claude 與怎麼選

API入口／重點常見特色
OpenAIAPI Platform → 帳務 → Key → Colab生態成熟、工具多
GeminiGoogle AI Studio 建金鑰，設 GEMINI_API_KEY，用 Google 官方 SDKGoogle 生態與多模態整合
ClaudeClaude Console／Platform 才是 API 入口（Claude.ai 是聊天產品）；設 ANTHROPIC_API_KEY，呼叫 Messages API長文、推理與文字工作常見
OpenRouter一把金鑰連多模型多模型比較與統一入口

常見 HTTP 問題

  • 401 金鑰錯誤；402／quota 帳務或額度；429 速率限制；模型名稱錯誤。
ACTION

8. 立即行動清單

  1. 先選一家 API，不要同時申請四家。
  2. 用最小 Colab 程式完成一次成功請求。
  3. 把 Key 移到 Secrets／環境變數。
  4. 查看 Token 使用量。
  5. 設定預算與上限。
  6. 再比較第二個模型的品質與成本。
「先讓一家 API 成功跑一次，再談比較；金鑰與預算，是每一步都要顧的兩件事。」
